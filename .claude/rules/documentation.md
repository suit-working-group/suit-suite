# Documentation Conventions

When editing or creating files in `docs/`:

## Language
- All documentation in English
- Professional, clear, concise style
- Acronyms defined at first use

## Glossary (`docs/claude-code-glossary.md`)
- Entries in alphabetical order
- Format: `### Term` → Definition → Context
- Self-contained definitions (understandable without slides)
- Update when new terms appear in slides

## References (`docs/references.md` — to be created)
- Organized by session number
- Format: Author (Year). *Title* → URL → Key points (2-3 bullets)
- Prioritize recent sources (2024-2026) for AI tools
- At least 1 academic + 1 practitioner source per topic

## General
- Use Markdown tables for structured data
- Use code blocks with language tags for examples
- Keep files focused — one concern per file
